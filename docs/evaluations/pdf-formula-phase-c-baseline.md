# PDF 实验性公式 Phase C：13 篇真实 PDF 基线评估

日期：2026-07-20

## 结论

当前实现能发现“含数学符号的文本行”，但尚不能稳定抽出“完整公式”。13 篇、196 页中共输出 157 个候选，其中 121 个（77%）触发自动残缺风险；对 181 个行尾编号公式代理，只命中 37 个（20%）。11 篇开启 `heuristic`，ResNet 与 BERT 整篇为 `none`。

问题核心不是单纯阈值偏高，而是评分前已经把二维公式拆成多行，且两个强信号在真实样本中失效：数学字体族无法从当前 pdf.js 文本项名称识别；双栏公式用整页中心判断，拿不到“居中编号”加分。直接把 `0.6` 降到 `0.55` 会把候选从 157 增至 450，但编号代理仅从 37 增至 54，噪声增长远快于有效召回。

## 方法与口径

- 语料：Attention、Adam、DDPM、VAE、ResNet、BatchNorm、LayerNorm、BERT、LoRA、DQN、GAN、PPO、Neural ODE；均为 arXiv 真实 PDF。
- 解析：`pdfjs-dist` 的 `getTextContent()`，复用生产的 `buildPageLines`、`stripHeadersFooters`、`detectHeading` 和 `detectPdfFormulaCandidates`；诊断器逐项复算当前评分。
- “编号代理”：正文中以 `(n)` 结尾的重建行。它不是人工公式真值，会漏掉编号被单独切行的公式，也可能包含表格项，因此只用于同口径相对比较。
- “残缺风险”：候选存在断裂边界、括号不平衡或邻接数学行；它是保守的自动风险标记，不等价于人工判错。
- Node 诊断出现标准字体资源告警；Attention 的 3 个候选及核心残缺文本已与真实 Edge 冒烟结果一致。字体对象结论限于当前 pdf.js 文本接口，不外推到所有 PDF 引擎。

## 残缺原因

1. **单行切割是首要原因。** `groupIntoLines` 以约 `0.45 × 字号` 的 y 差分行；分数、根号、上下标、求和上下限天然处于不同 y，先被拆开，随后每行独立评分。典型原始结果：

   - Attention 第 4 页：候选 `) = softmax( √ )V (1)`（1.00），前行 `QKT`，后行 `dk`。高分代表“信号强”，并不代表完整。
   - Attention 第 5 页：候选 `, xW1 +b1)W2 +b2 (2)`（0.80），前行 `FFN(x) = max(0`。
   - DDPM 第 2 页：`pθ(x0:T) := p(xT) pθ(xt−1|xt), pθ(`（0.67），相邻行是 `∏T`、`t=1`。
   - Neural ODE 第 4 页：`) = logp(z )−log∣det ∣ (6)`（0.80），相邻行仍有行列式和下标碎片。

2. **数学字体信号在这批样本中完全失效。** 13 篇的 `resolvedMathItemCount` 与 `internalMathItemCount` 均为 0；样式只暴露 `sans-serif/serif/monospace`，文本项名称是 `g_dN_fN`。现有 `cmmi|cmsy|stix|math...` 正则无法命中，因此所有候选实际只靠 Unicode、操作符和编号得分。

3. **分栏主要破坏居中门禁。** `centeredShort` 永远相对整页中心计算，没有按 `column=1/2` 使用栏中心。ResNet 的 `y = F(x,{Wi}) +x. (1)` 与 `y = F(x,{Wi}) +Wsx. (2)` 都只有 0.12，虽有编号却拿不到 0.68 的居中编号加分，最终整篇 `none`。DQN、GAN 等双栏样本也出现同型漏检。跨栏公式还可能在行重建前被分到左右两组，但本语料的直接证据以“栏中心误判”为主。

4. **编号与公式体的关系没有建模。** 编号附在残片尾部时会把残片推成高分；编号单独成行时可能只输出 `(2)`，也可能因不在整页中心而完全漏掉。当前没有用编号锚点向上、向下收拢同一公式块。

5. **字符语义确有不可逆损失。** DDPM 等样本出现控制字符、孤立根号和无法配对的装饰符。多行重组能恢复阅读顺序和更多 token，但仅靠文本层不能可靠恢复任意分数、矩阵、上下标归属与缺失字形。

## `0.6 / 0.65` 门禁影响

- 本地 `0.6` 是明显召回瓶颈，但它排除的不是“稍弱的完整公式”，而常是被拆开的公式片段。293 行落在 `0.55–0.59`；其中既有 `gt ← ∇θft(θt−1)`，也有普通正文、表格和局部符号。
- 全局 `0.65` 在这 13 篇中没有额外压掉任何文档：两篇 `none` 都是因为连一个 `>=0.6` 的本地候选也没有。由于当前有效分值离散地集中在 0.12、0.2、0.32、0.55/0.56、0.67/0.68 和 1.0，`0.65` 对本语料近乎冗余。
- 若仅降至 `0.55`：候选 157 → 450（+187%），编号代理命中 37 → 54（20% → 30%）。降至 `0.34`，编号代理也只到 60（33%）。结论：不能先降阈值，必须先形成公式块并重新定义“完整度”。

## LLM “先还原”是否掩盖缺陷

**会，而且当前成功推导不能作为抽取质量证据。** heuristic prompt 明确要求模型“还原为最可能的规范 LaTeX”；用户消息同时泄露论文标题、章节路径、页码、启发式分数，并由流水线加入该章节前 4 段正文（最多约 1500 tokens）。

Attention 的候选只剩 `) = softmax( √ )V (1)`，但标题、`Scaled Dot-Product Attention` 章节和正文已经给出 Q/K/V、点积及除以 `√dk`。经典论文记忆与上下文都足以补齐缺 token。即使模型输出正确，也无法区分“忠实还原”与“根据身份/语义猜中”。真正评估应把抽取完整度与推导质量分开，并对真实 Provider 做标题/章节/上下文消融；当前没有可用 Key，因此本报告不声称完成了模型侧盲测。

## 可改进点与上限

阶段 2 值得按以下顺序做低风险实验：

1. 以编号、操作符和几何邻近为锚点，先合并同页同栏的上下标/分数/续行，再评分；保留原行数组用于回滚和诊断。
2. 按栏宽与栏中心计算 `centeredShort`，把独立编号行与同一 y 带/邻近公式块关联。
3. 输出评分分解、块内原行和残缺风险，明确区分“检测置信度”与“文本完整度”。
4. 在块重建后才小幅校准阈值；以本语料的完整度、编号代理召回和噪声样本同时验收。

低风险启发式的合理上限是“显著改善常规 TeX 生成 PDF 的块级公式召回与 token 完整度”，不是可靠复原任意 PDF 的 LaTeX。若多行/分栏修复后仍大量存在缺字、矩阵与上下标歧义，应停止继续堆规则，进入阶段 3 技术路线选择。
