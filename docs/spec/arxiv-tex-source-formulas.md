# arXiv TeX 源公式优先：需求规格

## 背景与目标

Phase C 证明 PDF 文本层启发式可提高召回，但 13 篇真实样本的残缺风险仍显著，继续调阈值或叠加规则不能可靠恢复公式。下一步仅针对能识别 arXiv ID 的 PDF，尝试从同版本 arXiv TeX 源提取结构化公式；成功时替换 PDF 公式候选，失败时完整保留现有 PDF 正文与启发式公式结果。

## 范围

包含：版本化 arXiv ID、受限源文件获取、安全内存解包、主文件与 `input/include` 解析、章节和公式抽取、简单宏支持、来源标记、缓存升级、UI/导出/推导适配、13 篇语料验收。排除：执行或编译 TeX、OCR/云服务、通用网页找源、PDF 与 TeX 坐标对齐、修改 arXiv HTML/ar5iv 真 LaTeX 路径。

## 用户可见行为

源解析可靠且至少得到一个有效公式时，界面标记“arXiv TeX 源”，按真 LaTeX 渲染并走标准推导提示；不显示 PDF 页码或网页 DOM 回跳。源不存在、超限、损坏或解析无可靠公式时，显示可读降级原因并继续使用 Phase C 的“PDF 实验性公式”。

## 验收与风险

P0：任何 TeX 内容均不执行；路径穿越、链接、压缩炸弹和超限归档必须拒绝；源失败不能阻断 PDF 摘要；非 arXiv PDF、上传 PDF、arXiv HTML/ar5iv 行为不变。P1：Attention 的 scaled dot-product attention 与 FFN、ResNet 的核心编号公式可从源文本完整提取，且不触发“先还原 LaTeX”。P2：同一 13 篇语料记录源获取率、解析率、公式数、KaTeX 可渲染率和降级原因。主要风险是多文件工程、自定义宏与源码版本漂移；通过版本保留、受限宏表、确定性回退和可观测日志控制。
